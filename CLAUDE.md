# 100 Days Challenges

毎日ひとつ作品を作ってポートフォリオとして公開する、100日チャレンジのリポジトリです。

## 私について

- HTML/CSS/JS の初心者です。難しい用語は噛み砕いて説明してください。
- ビルドツールやフレームワークは使いません（React / Vite / npm などはなし）。
- ブラウザでファイルを直接開けば動く、素のHTML/CSS/JSだけを使います。

## 1日1作品のルール

- 成果物は **`day{N}.html` 1ファイル** に全部入れる（CSSもJSも `<style>` / `<script>` で内包する）。
  - 例: 今日が8日目なら `day8.html`
- 外部ファイルへの分割（`style.css` を別出しなど）は**しない**。
- 共通で使えるのは `favicon.svg` だけ。これは全ページで再利用する。
- 日本語UIで作る（`<html lang="ja">`）。
- スマホでも見られるようにレスポンシブにする（`@media (max-width: 640px)` を入れる）。

## 作品が完成したらやること

1. `index.html` のカード一覧（`<main class="grid">` の中）に、その日のカードを追加する。
   - 既存カードをコピーして、以下を書き換える:
     - `href="dayN.html"`
     - `data-category="tool" | "game" | "lp"`（カテゴリは3種類のみ）
     - `<span class="num">0N</span>`（番号は2桁ゼロ埋め）
     - `<span class="card-cat cat-XX">` のクラスとラベル（`cat-tool`/`cat-game`/`cat-lp`）
     - `<h2 class="card-title">作品名</h2>`
     - `<time datetime="YYYY-MM-DD">YYYY.MM.DD</time>`
   - 進捗バーとカテゴリ別カウントは **カード数から自動計算される** ので、数字を手で書き換える必要はない。
2. Git に1コミットして push する。コミットメッセージは英語1行で簡潔に（例: `Add Day 8: ...`）。

## カテゴリの基準

- **ツール (tool)**: 計算機・タイマー・診断・ToDoなど、使う系。
- **ゲーム (game)**: 遊ぶもの。
- **LP (lp)**: ランディングページ・架空ブランドのサイトなど、見せる系。

迷ったらユーザーに聞いてください。

## デザインの好み

- 既存ページの雰囲気を見ると分かりますが、ピンク〜ピーチ〜クリーム系の柔らかい配色が多めです。
- 各作品ごとに世界観を変えてOK（毎日違うテイストにする）。
- フォントは Google Fonts を `<link>` で読み込んで使う。

## やらないこと

- 過去の `dayN.html` を勝手にリファクタしない（必要なときだけ、聞いてから直す）。
- READMEやドキュメントを勝手に増やさない。
- `index.html` のレイアウトやスタイル設計を勝手に変えない（カード追加だけにとどめる）。
